About TypeScript

TypeScript is a programming language developed by Microsoft that extends JavaScript by adding static types, making it easier to build large-scale applications with fewer errors. Created by Anders Hejlsberg (the architect of C#), TypeScript was first released in 2012 to address some of the shortcomings of JavaScript when used for complex applications.

JavaScript is a dynamic language, meaning variables and functions can accept any type of data, which can lead to unexpected errors and bugs. TypeScript introduces a type system that allows developers to explicitly define types for variables, function parameters, and return values, providing more predictability and safety in the code.

The main goal of TypeScript is to enhance the developer experience by providing tools for better code organization, readability, and maintainability, especially in large projects where JavaScript alone can become difficult to manage. TypeScript compiles to plain JavaScript, which means it can run in any environment that supports JavaScript, such as web browsers and Node.js.

Over the years, TypeScript has gained significant popularity in the development community due to its ability to catch errors early, support modern JavaScript features (like ES6 and beyond), and integrate seamlessly with popular development tools like Visual Studio Code.

TypeScript's growth has been fueled by the increasing complexity of web applications and the demand for more robust tooling around JavaScript. Major frameworks like Angular have adopted TypeScript as their primary language, and many other frameworks, such as React and Vue, offer first-class support for it.

Today, TypeScript is widely used by developers and companies worldwide to build scalable, maintainable, and reliable applications, and it continues to evolve as a standard choice for JavaScript developers looking for better control and safety in their codebases.

Basic TypeScript

TypeScript is a programming language that builds on JavaScript by adding static types, making it easier to catch errors and write more reliable code. Think of it as a "superset" of JavaScript, meaning it includes everything JavaScript can do, but with some extra features.

The most important feature is "type safety," which allows you to specify the types of variables, function parameters, and return values, ensuring that your code behaves as expected. This is especially helpful in large projects where understanding what kind of data is being used can be tricky. TypeScript code is then "transpiled" into regular JavaScript so that it can run in any environment that supports JavaScript, like web browsers or Node.js.

Overall, TypeScript helps developers write cleaner, more maintainable code and catch mistakes early in the development process.


Basic Types

TypeScript allows you to define types for variables.

Examples:


            let message: string = 'Hello, TypeScript!';
            let count: number = 5;
            let isActive: boolean = true;

Functions

You can define functions with type annotations to specify the types of parameters and return values.

Example:


            function add(a: number, b: number): number {
              return a + b;
            }
            let result: number = add(3, 5);

Interfaces

Interfaces define the shape of objects.

Example:


            interface Person {
              name: string;
              age: number;
            }
            let person: Person = { name: 'Eleanor', age: 25 };

Classes

TypeScript supports classes and object-oriented programming.

TypeScript checks that name is always a string. If you try to create an Animal with a non-string name, or assign a non-string value to name, TypeScript will report an error at compile time.

Example:


            class Animal {
              name: string;
              constructor(name: string) {
                this.name = name;
              }
              speak() {
                console.log(this.name + ' makes a noise.');
              }
            }
            let cat = new Animal('Kitty');
            cat.speak();


            --- Console log output: Kitty makes a noise. ---

Setting up a TypeScript project

Step 1 - Set Up Project Directory

Create a new directory for your TypeScript project and navigate into it. You can do this using the terminal: 


            mkdir my-typescript-project
            cd my-typescript-project

Step 2 - Install Node.js and npm

To check if you have Node.js and npm installed, run the following commands in your terminal or command prompt:


            node -v
            npm -v

If you don't have Node.js and npm installed, you can download and install them from the official Node.js website.


Step 3 - Initialise package.json File

To initialise a new Node.js project, run the following command in your terminal: 


            npm init -y

This will create a package.json file in your project directory with default settings.


Step 4 - Install TypeScript and Initialise Config File

To install TypeScript, run the following command in your terminal:


            npm install typescript --save-dev

Next, you need to initialise a TypeScript configuration file by running:


            npx tsc --init

Step 5 - Create TypeScript File

Create a new TypeScript file in the Source folder in your project directory. The folder should be called "src", and the TypeScript file should have a ".ts" extension.


Step 6 - Add Scripts to package.json

Open the package.json file in your project directory and add the following scripts to the "scripts" section: 


            "scripts": {
              "build": "tsc",
              "start": "node dist/index.js",
              "watch": "tsc --watch"
            }

  • npm run build - will compile the TypeScript files.
  • npm start - will run the compiled JavaScript.
  • npm run watch - watch will watch for file changes and recompile automatically.


Step 7 - Add Your Code

Write your TypeScript code in the TypeScript file you created earlier. You can use any text editor or IDE to write TypeScript code.


Step 8 - Run Your Code!

Run your code using the commands in Step 6. You should see the output of your TypeScript code in the terminal. 


            npm run build
            npm start

JavaScript vs TypeScript

JavaScript is a loosely typed language, which means it can be challenging to determine the types of data being used. In JavaScript, there is no built-in information about the types of function parameters or variables, so developers often have to rely on documentation or make educated guesses based on the code.

TypeScript, on the other hand, allows developers to explicitly define data types within their code. This feature helps catch errors when there is a mismatch between expected and actual types. For instance, TypeScript will throw an error if you try to pass a string to a function that expects a number, whereas JavaScript will not.

Here are some examples of the differences between JavaScript and TypeScript:


Functions

JavaScript:


              function add(a, b) {
                return a + b;
              }
              
              console.log(add(5, 10));   -- 1 --
              console.log(add("5", 10));   -- 2 --


              1 -- Console log output: 15
              2 -- Console log output: "510"

TypeScript:


              function add(a: number, b: number): number {
                return a + b;
              }
              
              console.log(add(5, 10));    -- 1 --
              console.log(add("5", 10));   -- 2 --


              1 -- Console log output: 15
              2 -- Error: Argument of type 'string' is not assignable to parameter of type 'number'.

In JavaScript, the add function can accept any types for a and b. This can lead string concatenation instead of addition of the numerical values.

In TypeScript, the function specifies that a and b must be of type number. If you try to pass a string instead, TypeScript will report an error.


Array Types

JavaScript


              const numbers = [1, 2, 3];
              numbers.push("four");

              console.log(numbers);
            

              --- Console log output: [1, 2, 3, "four"] ---

TypeScript


              const numbers: number[] = [1, 2, 3];
              numbers.push("four");

              console.log(numbers);


              --- Console log output: [1, 2, 3] ---

JavaScript allows mixed types in an array without any warnings, potentially leading to unexpected behaviors.

TypeScript enforces that the numbers array should only contain numbers. Pushing a string to the array will result in a compile-time error.


Object Properties

JavaScript


              const user = {
                name: "Eleanor",
                age: 25,
              };
              
              console.log(user.name);   -- 1 --
              console.log(user.email);   -- 2 --
              

              1 -- Console log output: Eleanor
              2 -- Console log output: undefined

TypeScript


              type User = {
                name: string;
                age: number;
              };
              
              const user: User = {
                name: "Eleanor",
                age: 25,
              };
              
              console.log(user.name);   -- 1 --
              console.log(user.email);   -- 2 --


              1 -- Console log output: Eleanor
              2 -- Console log output: Error: Property 'email' does not exist on type 'User'.

JavaScript doesn't enforce any constraints on object properties, which can lead to accessing properties that don't exist without any errors.

TypeScript defines a User type with specific properties. Accessing a property that doesn't exist will cause a compile-time error.
